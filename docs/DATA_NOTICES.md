# 공지 데이터(JSON) 사용법 (서버 없이)

이 사이트는 서버/DB 없이도 공지 목록을 **JSON 파일**로 관리할 수 있습니다.

## 파일 위치
- `data/notices.json`

## 데이터 형식
각 공지는 아래 필드를 가집니다.

- `id` : 고유 ID (추천: `n-YYYY-MM-DD-01` 형식)
- `category` : 분류 (예: `서버점검`, `업데이트`, `이벤트`, `서버규정`)
- `title` : 제목
- `date` : 날짜 (`YYYY-MM-DD`)
- `summary` : 요약 (목록에서 보임)
- `content` : 본문 (상세에서 보임, 줄바꿈은 `\n`)

예시:
```json
{
  "id": "n-2026-01-15-01",
  "category": "업데이트",
  "title": "메뉴 UX 최종 정리 반영",
  "date": "2026-01-15",
  "summary": "드롭다운 UX/접근성/운영 안정화 반영.",
  "content": "- 항목1\n- 항목2"
}
```

## 페이지
- 목록: `pages/공지/index.html`
- 상세: `pages/공지/notice_view.html?id=공지ID`

## 운영 팁
- 새 공지를 추가한 후에는 `notices.json`을 저장하고 업로드만 하면 됩니다.
- 목록은 `date` 기준으로 최신순 자동 정렬됩니다.
- 분류는 자동으로 필터에 반영됩니다.
