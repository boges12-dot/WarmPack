# STEP21 (2026-01-17) - pages 경로 UTF-8 파일명 정리

## 문제
Windows에서 ZIP을 만들거나 특정 도구로 압축/해제할 때, `pages/공지` 같은 한글 폴더/파일명이
리눅스(또는 일부 호스팅) 환경에서 **깨진 이름(모지바케)** 으로 풀리면서 링크가 404가 되는 문제가 발생할 수 있습니다.

## 조치
- `/pages` 하위 폴더명을 다음과 같이 **정상 한글 이름으로 통일**했습니다.
  - 공지, 가이드, 제작, 클래스, 스킬, 아이템, 사냥터, 홍보, 커뮤니티, 다운로드
- 각 섹션의 하위 페이지 파일명도 `index.html` 메뉴 링크와 동일하게 통일했습니다.
  - 예: `pages/공지/서버규정.html`, `pages/스킬/군주-스킬.html` 등

## 확인 방법
1. 로컬에서 `site/index.html` 열고 메뉴 클릭이 모두 정상 이동되는지 확인
2. GitHub Pages 등 리눅스 기반 호스팅에 올린 뒤, 메뉴 링크 404 발생 여부 확인
